# 樂透刮刮樂吉祥卡 (Lottery Scratch Card)

## 1. 專案描述
開發一個靜態網頁形式的「樂透刮刮樂吉祥卡」，目標部署於 GitHub Pages。內容為數位刮刮樂，使用者刮開卡片後可獲得隨機的馬年吉祥話。
考量使用者多透過行動裝置訪問，網頁必須採用 Responsive Design (RWD) 並針對手機觸控體驗最佳化。

須滿足 Android, iOS 的兼容性，主要會透過 linebot 裡面的 WebView 來訪問，因此需要確保在這些平台上的顯示和功能正常。

## 2. 功能需求
- **刮刮樂功能**：使用者可以透過滑動或點擊的方式刮開卡片，揭示隱藏的內容。
- **隨機吉祥話生成**：每次刮開卡片後，系統會隨機生成一條與馬年相關的吉祥話。
- **響應式設計**：網頁需適應不同尺寸的螢幕，確保在手機和平板上都有良好的使用體驗。
- **兼容性**：確保在 Android 和 iOS 平台上的瀏覽器中都能正常運行，特別是在 linebot 的 WebView 中。
- **視覺設計**：卡片的設計應該具有吸引力，符合馬年主題，並且提供愉悅的使用體驗。並加入一些企鵝元素for 401班級特色，可參考 ![alt text](test.jpg) ![alt text](charater.jpg) ![alt text](backgroud.jpg) 這些圖片作為設計靈感。
- **社交分享功能**：使用者可以將刮開的吉祥話分享到社交媒體平台，如 Facebook、Twitter 等。
- **多語言支持**：提供至少中文和英文兩種語言選項，讓更多使用者能夠享受這個刮刮樂體驗。
- **使用者介面**：設計簡潔直觀的使用者介面，讓使用者能夠輕鬆理解如何操作刮刮樂。
- **性能優化**：確保網頁載入速度快，刮刮樂的互動過程流暢，提供良好的使用體驗。
- **安全性**：確保網頁的安全性，防止潛在的攻擊和漏洞，保護使用者的資料和隱私。
- **測試和調試**：在開發過程中進行充分的測試和調試，確保所有功能正常運作，並且在不同設備和瀏覽器上都能兼容。
- **部署和維護**：將完成的網頁部署到 GitHub Pages 上，並定期進行維護和更新，確保其持續運行和提供良好的使用體驗。
- **用戶反饋機制**：提供一個簡單的反饋機制，讓使用者可以提交他們的意見和建議，以便未來的改進和優化。
